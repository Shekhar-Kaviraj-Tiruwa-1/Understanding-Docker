Concept 01

What is Docker?

Docker lets you package an app and everything it needs into a self-contained box called a container. It runs the same way everywhere ‚Äî your Mac, your Raspberry Pi, a cloud server.

üì¶
The Analogy
Shipping Container
Before shipping containers, cargo had to be individually packed onto each ship in different ways. Docker containers work the same way ‚Äî pack your app once, ship it anywhere. The ship (your computer) doesn't care what's inside.
üç±
Another Way to Think of It
Packed Lunchbox
A container is like a lunchbox ‚Äî it has everything the app needs packed inside: the food (code), the utensils (libraries), and the napkin (config). Open any lunchbox and it works the same.
üè†
Without Docker
The "Works on My Machine" Problem
App works on your laptop. Fails on the server. Fails on your Raspberry Pi. Everyone has different versions of Python, Node, libraries. Docker eliminates this ‚Äî the container includes everything.
üß±
Key Concept
Image vs Container
An Image is the recipe (read-only blueprint). A Container is the meal you made from that recipe (a running instance). One image can spawn many containers simultaneously.

Comparison 04

Docker vs Virtual Machine

Both Docker and VMs isolate software ‚Äî but they work very differently. Since you're using UTM to run Ubuntu, this comparison is especially relevant to you.

Virtual Machine (UTM)
üñ•Ô∏è Full OS, heavy weight
  • Boots a complete operating system (Ubuntu, Windows, etc.)
  • Needs its own RAM allocation (you gave 4GB to Ubuntu)
  • Startup takes 30‚Äì60 seconds ‚Äî full OS boot
  • Takes gigabytes of disk space per VM
  • Has its own kernel ‚Äî full isolation from host
vs
Docker Container
üì¶ Shares kernel, lightweight
  • Shares the host OS kernel ‚Äî no full OS needed
  • Uses only the RAM the app actually needs (often 50‚Äì200MB)
  • Starts in milliseconds ‚Äî no OS boot
  • Image is megabytes, not gigabytes
  • Process-level isolation ‚Äî isolated but lightweight

Your home lab strategy: Use UTM (VM) to run Ubuntu on your Mac Mini as the foundation. Then run all your home lab services inside Docker containers within that Ubuntu VM. You get the best of both ‚Äî a proper Linux environment from the VM, and lightweight, easy-to-manage services from Docker.
üöÄ
Real example ‚Äî running Pi-hole in Docker:

docker run -d --name pihole \
  -p 53:53/tcp -p 53:53/udp -p 80:80 \
  -e TZ="America/Toronto" \
  -v pihole_data:/etc/pihole \
  --restart unless-stopped \
  pihole/pihole:latest

This one command downloads and starts Pi-hole ‚Äî no manual installation needed. The -v flag saves its data permanently so it survives container restarts.
